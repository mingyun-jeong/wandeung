# 완등 UI/UX 전면 개선 디자인 플랜

> 작성일: 2026-03-14
> 상태: 설계 (Design)

## 현재 상태 분석

현재 앱은 기본적인 기능(촬영, 기록, 캘린더, 통계)이 잘 갖춰져 있으나, 다음과 같은 UX 개선 여지가 있음:

| 영역 | 현재 상태 | 주요 이슈 |
|------|-----------|-----------|
| 홈 탭 | 30일 통계 카드 + 최근 암장 마퀴 + 최근 기록 리스트 | 정보 밀도 낮음, 동기부여 요소 부족 |
| 기록 탭 | TableCalendar + 필터바 + 리스트 | 필터 UX 복잡, 날짜별 성과 시각화 없음 |
| 카메라 탭 | 풀스크린 카메라 + 오버레이 | 촬영 전 메타데이터 설정이 작고 눈에 안 띔 |
| 암장 난이도 탭 | 검색 + 확장 가능 카드 리스트 | 시각적으로 단조로움 |
| 통계 탭 | 기간 칩 + 요약 카드 + 3개 차트 | 차트 가독성, 인사이트 부족 |
| 기록 상세 | 비디오 + 메타데이터 | 레이아웃 밋밋, 공유 기능 없음 |
| 기록 저장 | 비디오 프리뷰 + 폼 필드 | 폼이 길어 스크롤 필요, 단계 구분 없음 |
| 전반적 테마 | Teal 기반 Material 3 | 클라이밍 앱 느낌 부족, 브랜드 아이덴티티 약함 |

## Phase 1: 테마 및 브랜드 아이덴티티 강화

### 1.1 컬러 시스템 리뉴얼

**목표:** 클라이밍의 역동적이고 에너지 넘치는 느낌을 반영하는 컬러 팔레트

```
현재:  Primary #14B8A6 (Teal)
       Background #F8FAFB
       Border #E8ECF0

개선안:
  Primary     → #1A1A2E (Deep Navy) — 앱 바, 주요 텍스트
  Accent      → #E94560 (Climbing Red) — FAB, CTA 버튼, 완등 표시
  Secondary   → #0F3460 (Mid Navy) — 보조 텍스트, 카드 헤더
  Success     → #16C784 (Green) — 완등 성공 상태
  InProgress  → #F5A623 (Amber) — 도전중 상태
  Surface     → #F5F5F7 (Warm Gray) — 배경
  Card        → #FFFFFF — 카드 배경

  Gradient (CTA):  #E94560 → #FF6B6B (따뜻한 그라디언트)
```

**이유:** 현재 Teal은 클라이밍과 연관성이 약함. 다크 네이비 + 레드 조합은 암벽의 느낌과 도전적인 에너지를 표현.

### 1.2 타이포그래피 개선

```dart
// 현재: 기본 Material 폰트
// 개선: Pretendard 또는 SUIT 폰트 적용 (한국어 최적화)

headlineLarge:  24px, Bold  — 화면 타이틀
headlineMedium: 20px, Bold  — 섹션 헤더
titleMedium:    16px, SemiBold — 카드 타이틀
bodyLarge:      15px, Regular — 본문
bodyMedium:     14px, Regular — 보조 텍스트
labelSmall:     12px, Medium — 뱃지, 태그
```

### 1.3 앱 아이콘 & 스플래시 리뉴얼

- 앱 아이콘: 산/암벽 실루엣 위에 깃발 모티프 (완등 = 정상 등정)
- 스플래시: 현재 애니메이션 유지하되, 새 컬러 팔레트 적용

## Phase 2: 홈 탭 개선 — 동기부여 대시보드

### 2.1 히어로 카드 → 오늘의 클라이밍 요약

```
┌─────────────────────────────────────┐
│  🔥 연속 3일째 클라이밍 중!           │
│                                     │
│  이번 주 완등률                       │
│  ████████████░░░  78%               │
│                                     │
│  완등 12개  ·  도전중 4개  ·  총 16개  │
│                                     │
│  [오늘 기록하기 →]                    │
└─────────────────────────────────────┘
```

**변경점:**
- 연속 기록(streak) 표시로 동기부여
- "오늘 기록하기" CTA 버튼 추가
- 기간이 아닌 "이번 주" 단위로 친숙하게

### 2.2 최근 기록 → 카드 그리드 (2열)

```
┌──────────┐ ┌──────────┐
│ 📹 썸네일  │ │ 📹 썸네일  │
│ ● 빨강 V3 │ │ ● 파랑 V5 │
│ 완등 ✓    │ │ 도전중    │
│ 에어즈락   │ │ 에어즈락   │
└──────────┘ └──────────┘
```

**변경점:**
- 현재 리스트 → 2열 그리드로 시각적 밀도 증가
- 썸네일 중심 레이아웃 (비디오 앱 느낌)
- 컬러 도트 + 등급이 한눈에 보이게

### 2.3 주간 활동 히트맵 (미니)

```
월  화  수  목  금  토  일
◌   ●   ●   ◌   ◌   ●   ─
          (오늘)
```

- 이번 주 어느 날 클라이밍했는지 한눈에 표시
- 채워진 원(●)은 기록 있는 날, 빈 원(◌)은 없는 날
- 오늘 미기록 시 은근한 동기부여

## Phase 3: 기록(캘린더) 탭 개선

### 3.1 캘린더 날짜 셀 개선

```
현재:  날짜 아래 작은 점만 표시
개선:  날짜 셀 안에 미니 컬러 도트 클러스터
```

```
┌────┐
│ 14 │
│●●●○│  ← 완등 3개(채움), 도전중 1개(빈)
└────┘
```

- 각 날짜에 기록 수와 완등/도전 비율을 시각적으로 표현
- 날짜 선택 시 부드러운 확대 애니메이션

### 3.2 필터 UX 개선

```
현재:  가로 스크롤 필터 칩 (PopupMenu)
개선:  필터 시트 (BottomSheet)
```

- 필터 아이콘 버튼 하나로 통합
- BottomSheet에서 모든 필터를 한 번에 설정
- 적용된 필터는 상단에 작은 칩으로 표시 + "초기화" 버튼
- 필터 적용 수 뱃지: `필터 (3)`

### 3.3 기록 리스트 → 스와이프 액션

- 왼쪽 스와이프: 삭제 (빨간 배경 + 휴지통)
- 오른쪽 스와이프: 편집 (파란 배경 + 연필)
- Dismissible 위젯 활용

## Phase 4: 카메라 탭 UX 개선

### 4.1 촬영 전 메타데이터 선택 개선

```
현재:  작은 오버레이 칩 (좌측)
개선:  하단 카드 형태의 메타 패널
```

```
┌─────────────────────────────────┐
│        [카메라 프리뷰]            │
│                                 │
│                                 │
│                                 │
│   ┌───────────────────────┐     │
│   │ 에어즈락 강남 ▼  📍    │     │
│   │ ● 파랑 (V4~V6) ▼     │     │
│   └───────────────────────┘     │
│                                 │
│        ( ⏺ REC )               │
└─────────────────────────────────┘
```

**변경점:**
- 메타데이터 패널을 하단에 배치 (엄지 접근성)
- 현재 선택된 암장과 난이도가 명확하게 보임
- 촬영 중에는 자동 축소/숨김

### 4.2 촬영 후 플로우 간소화

```
현재:  촬영 → RecordSaveScreen (긴 폼)
개선:  촬영 → 퀵 저장 시트 (필수 정보만) → 상세 편집은 선택
```

- 난이도/암장은 이미 카메라에서 선택 → 자동 반영
- 퀵 저장: 상태(완등/도전중) 선택 → 바로 저장
- "상세 입력" 토글로 태그, 메모 등 추가 가능
- 저장 후 바로 홈으로 (기록 자동 새로고침)

### 5.1 몰입형 비디오 뷰

```
┌─────────────────────────────────┐
│  [비디오 플레이어 — 풀 너비]       │
│                                 │
│  ← 뒤로     ⋮ 메뉴              │
│                                 │
│        ▶ 재생                   │
│                                 │
│  ● 파랑  완등 ✓                  │
│  에어즈락 강남   2026.03.14       │
└─────────────────────────────────┘
│                                 │
│  태그: #슬랩 #밸런스             │
```

**변경점:**

- 공유 버튼 추가 (비디오 + 메타데이터를 이미지로 내보내기)

### 5.2 같은 루트 히스토리

- 동일 암장 + 동일 난이도의 과거 기록 보기
- "이 루트 도전 기록" 섹션으로 성장 과정 확인

## Phase 6: 통계 탭 개선

### 6.1 인사이트 카드 추가

```
┌─────────────────────────────────┐
│ 💪 이번 달 인사이트               │
│                                 │
│ • 완등률이 지난달 대비 12% 상승!   │
│ • 가장 많이 간 암장: 에어즈락 (8회) │
│ • 최고 난이도 달성: V6 파랑 완등   │
└─────────────────────────────────┘
```

- 단순 숫자 나열이 아닌, 자연어 인사이트 제공
- 성장을 체감할 수 있는 비교 문구

### 6.2 난이도 분포 → 레이더 차트 or 스택 바

```
현재:  다중 라인 차트 (가독성 낮음)
개선:  스택 바 차트 (날짜별 난이도 분포)
```

```
3/1  ███░░░░░  빨강3 + 파랑1
3/2  █████░░░  노랑2 + 초록2 + 파랑1
3/3  ██░░░░░░  파랑2
```

- 각 날짜별 난이도 색상 스택으로 한눈에 파악
- 시간에 따라 색상 분포가 어두운 쪽(고난이도)으로 이동하는 것을 시각적으로 확인

### 6.3 개인 최고 기록 뱃지

- "최초 V5 완등", "10일 연속 기록" 등 마일스톤 뱃지
- 달성 시 축하 애니메이션 (confetti)

## Phase 8: 접근성 & 사용성

### 8.1 빈 상태(Empty State) 개선

```
현재:  아이콘 + 텍스트
개선:  일러스트 + 안내 텍스트 + CTA 버튼
```

- 첫 기록이 없을 때: 클라이머 일러스트 + "첫 클라이밍을 기록해보세요!" + [촬영하기] 버튼
- 필터 결과 없을 때: "조건에 맞는 기록이 없어요" + [필터 초기화] 버튼

### 8.2 온보딩 플로우 (최초 실행)

- 3페이지 온보딩 스와이프:
  1. "클라이밍을 촬영하세요" — 카메라 아이콘
  2. "난이도와 상태를 기록하세요" — 컬러 그리드
  3. "성장 과정을 확인하세요" — 차트
- PageView + smooth_page_indicator

### 8.3 터치 타겟 표준화

- 모든 인터랙티브 요소 최소 48x48dp
- 현재 38px인 재생 버튼 등 수정

## 기술적 고려사항

1. **테마 변경 시**: `app.dart`의 ThemeData만 수정하면 Material 컴포넌트는 자동 반영. 커스텀 하드코딩된 색상(`Color(0xFF...)`)은 개별 수정 필요.
2. **홈 탭 그리드**: `SliverGrid` 사용으로 스크롤 성능 확보
3. **카메라 오버레이 변경**: 기존 `CameraGradeOverlay`, `CameraGymOverlay` 위젯을 하단 패널로 통합
4. **캘린더 셀 커스텀**: `TableCalendar`의 `calendarBuilders` 파라미터로 커스텀 셀 렌더링
5. **퀵 저장**: `RecordSaveScreen`을 리팩터링하여 `QuickSaveSheet` (BottomSheet)와 `DetailSaveScreen`으로 분리
